网络编程：zmq学习笔记

ZMQ 简介

ØMQ、ZeroMQ简称ZMQ：一个简单的传输层，一个消息处理队列库
MQ：Message Queue
Device:实现路由功能的组件
ZMQ:更像一个底层的网络通讯库，在Socket API之上做了一层封装，将网络通讯、进程通讯和线程通讯抽象为一个统一的API接口

ZMQ 优势

ZMQ与Socket区别：普通的socket是端到端的(1:1的关系)，而ZMQ却是N:M的关系，ZMQ屏蔽了点到点之间需要显示的建立连接、销毁链接、选择协议和处理错误等细节，让网络编程更为简单。ZMQ用于node与node间的通信，node可以是主机或者是进程。

所有的网络交互所使用的API实际上是Berkeley套接字（BSD）。
ZeroMQ网络库 能对各种套接字类型、连接处理、帧、甚至路由的底层细节进行抽象。
提供一些跨多种传输协议（如进程内通讯、IPC、TCP和广播）的套接字供你使用。你可使用多种方式实现N对N的套接字连接，譬如：扇出、发布订阅、任务分发以及请求响应。

ZMQ是网络栈中新的一层，它是个可伸缩层，分散在分布式系统间。可支持任意大的应用程序。ZMQ不是简单的点对点交互，相反，它定义了分布式系统的全局拓扑。ZMQ应用程序没有锁，可并行运行。此外，它可在多个线程、内核和主机盒之间弹性伸缩。

ZMQ提供了三个基本的通信模型

1、Request-Reply
2、Publisher-Subscriber
3、Parallel Pipeline

Request-Reply通信模式

说明：
1)、服务器端和客户端无论谁先启动，效果是相同的，和socket不一样
2)、在服务器端收到信息之前，程序是阻塞的，会一直等待客户端连接上来
3)、ZMQ通信通信单元是消息，他除了知道Bytes的大小，他并不关心消息的格式，可以使用任何数据格式如　　：XML，Protocol Buffers、Thrift、json等等
4)、Socket类型：SOCKET_REP和SOCKET_REQ类型

Publish-Subscribe通信模式

由一个节点提供信息源，由其他的节点，接受信息源的信息

说明：
1).Socket类型：SOCKET_PUB和SOCK_SUB
2).客户端需要$subscriber->setSockOpt(ZMQ::SOCKOPT_SUBSCRIBE,$filter);设置一个过滤值，相当于设定一个订阅频道，否则任何消息也收不到
3).服务器端一直不断的广播中，如果中途有Subscriber端退出，并不影响他继续广播，当Subscriber再连接上来的时候，收到的就是后来发送的新的信息。这对比较晚加入的，或是中途离开的订阅者，必然会丢掉一部分信息，这是这个模式的一个问题，所谓Slow joiner。
4).如果Publisher中途离开，所有的Subscriber会hold住，等待Publisher再上线的时候，会继续接受信息。

PipeLine模型

场景应用：如果需要统计各个机器的日志，我们需要将统计任务分发到各个节点机器上，最后手机统计结果，做一个汇总。

说明：
1)、Socket类型：ZMQ::SOCKET_PUSH，ZMQ::SOCKET_PULL,
2)task ventilator上游 使用SOCKET_PUSH把任务分发到Worker节点上，而Worker节点使用SOCKET_PULL从上游接受任务，并使用SOCKET_PUSH将结果汇集到Slink
3)任务的分发的时候也同样有一个负载均衡的路由功能，worker可以随时自由加入，task ventilator可以均衡将任务分发出去

其他扩展模式

通常，一个节点，既可作为Server，同时也能作为Client，通过PipeLine模型中的Worker,他向上连接着任务分发，向下连接着结果搜集的Sink机器。可以利用这种特性，丰富的扩展原有的三种模式。

利用ZeroMQ实现一个配置推送中心
当我们将WEB代码部署到集群上的时候，如果需要实时的将最新的配置信息，主动的推送到各个机器节点。在此过程中，我们一定要保证，各个节点收到的信息的一致性和正确性，如果使用HTTP,由于他的无状态性，我们无法保证信息的一致。

如果使用ZMQ的信息订阅模式。在这个模式中，对于后来的加入节点，始终会丢失在他加入之前，已经发送的信息（Slow joiner）我们可以开启另外一个ZMQ的通信通道，用于报告当前节点的情况（节点的身份、准备状态等）。

通过$context->getSocket(ZMQ::SOCKET-REQ)设置一个新的Request-Reply连接，来用于Subscriber向Publisher报告自己的身份信息，而Publisher则等待Subscriber都连接上的时候，再选择Publish自己的信息

网络编程：socket学习笔记

套接字描述符（socket descriptor）可以理解为一种文件描述符。

  • 套接字种类：
    1、DARPA Internet地址（Internet套接字）
    2、本地节点的路径名（Unix套接字）
    3、CCITT X.25地址（X.25套接字）

  • Internet套接字：
    1、Stream Sockets(流格式) SOCK_STREAM 可靠的双向通讯的数据流
    2、Datagram Sockets(数据报格式) SOCK_DGRAM 无连接套接字
    3、Raw Sockets

  • 回顾网络名词
    1、TCP(The Transmission Control Protocol): 传输控制协议
    2、IP(Internet Protocol): 因特尔协议：处理Internet路由
    3、UDP(User Datagram Protocol): 用户数据报协议 应用：tftp,bootp
    4、ACK

  • 回顾网络七层协议
    7、应用层(Application)——————-应用层:telnet,ftp
    6、表示层(Presentation)
    5、会话层(Session)
    4、传输层(Transport)———————传输层(Host-to-Host Transport Layer)(TCP,UDP)
    3、网络层(Network)———————-网络访问层(Network Access Layer)
    2、数据链路层(Data Link)
    1、物理层(Physical)
  • socket通信基本数据结构

  • 网络字节顺序和本机字节顺序之间的转换：
    1、htons(): Host to Network Short
    2、htonl(): Host to Network Long
    3、ntohs(): Network to Host Short
    4、ntohl(): Network to Host Long

  • 基本常量和基本函数

socket()函数

说明：
domain:AF_INET
type:告诉内核是SOCK_STREAM类型还是SOCK_DGRAM
protocol 设置为0
返回socket描述符

bind()函数

说明：
功能：讲套接字和机器上的一定的端口关联起来
sockfd: 调用socket返回的文件描述符
my_addr:指向数据结构struct sockaddr的指针 它保存你的地址和端口信息
addrlen设置为sizeof(struct sockaddr)
错误时返回-1

端口：
所有小于1024的端口号都被系统保留，可以选择1024到65535的端口

connect()函数

说明：
sockfd:系统调用socket()返回的套接字文件描述符
serv_addr保存着目的地端口和IP地址的数据结构struct sockaddr
addrlen设置为sizeof(struct sockaddr)

listen()函数

说明：
sockfd是调用socket()返回的套接字文件描述符
backlog是在进入队列中允许的连接数目
返回 发生错误的时候返回-1，并设置全局错误变量errno

accept()函数

说明：
addr指向局部的数据结构sockaddr_in的指针
addrlen局部的整型变量 sizeof(struct sockaddr_in)
在错误时返回-1，并设置全局错误变量errno
功能：
有人从很远的地方通过一个你在侦听(listen())的端口连接(connect())到你的机器。它的连接将加入到等待接受(accept())的队列中。你调用accept()告诉它你有空闲的连接。它将返回一个新的连接字文件描述符。这样你就有两个套接字了，原来的一个还在监听你的那个端口，新的在准备发送(send())和接受(rev())数据。

send()和recv()函数

功能：
这两个函数用于流式套接字或者数据包套接字的通讯。
说明：
sockfd是你想发送数据的套接字描述符(或者是调用socker()或者是accept()返回的)
msg是指向你想发送的数据的指针
len是数据的长度
flag设置为0
返回实际发送的数据的字节数(它可能小于你要求发送的数目)，错误时候返回-1，并设置errno

说明：
sockfd是要读的套接字描述符。
buf是要读的信息的缓冲
len是缓冲的最大长度
flags设置为0

sendto()和recvfrom()

无连接的数据报套接字发送接受消息

说明：
to是指向数据结构struct sockaddr的指针,包含了目的地址的ip地址和端口信息
tolen可以简单的设置为sizeof(struct sockaddr)

说明：
from是一个指向局部数据结构struct sockaddr的指针，它的内容是源机器的ip地址和端口信息
fromlen是个int型的局部指针 初始值为sizeof(struct sockaddr) 函数调用返回后，fromlen保存着实际储存在from中的地址的长度

close(sockfd);

功能：
关闭你的套接字描述符

int shutdown(int sockfd,int how);

功能：
允许将一定方向上的通讯或者双向的通讯关闭
说明：
how:0 不允许接受
1 不允许发送
2 不允许接受和发送和close一样

getpeername()函数

功能：
返回你程序所运行的机器的主机名字
说明：
addr:保存着连接的另一边的信息
addlen初始化为sizeof(struct sockaddr)

gethostname()函数

功能：
返回时保存主机名
说明：
hostname 保存主机名
size hostname数组的字节长度

gethostbyname();

功能：
返回域名对应的主机ip
